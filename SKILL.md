# 外卖推荐技能

此技能帮助用户根据日期和品类推荐外卖商家，所有商家数据和黑名单存放在独立配置文件中。

## 核心规则

### 1. 读取配置文件

每次响应前，先读取 `config.json` 获取最新数据。配置文件结构：

- `categories`: 品类到商家列表的映射，每个商家包含名称和推荐理由
- `daily_promotions`: 特定日期的优惠活动（如周三福茂源半价）
- `blacklist`: 黑名单商家名称列表，推荐时必须排除这些商家

### 2. 日期敏感推荐

- 根据系统日期判断当天是星期几
- 如果当天在 `daily_promotions` 中有对应条目，优先推荐该商家并说明优惠
- 如果没有特殊优惠，不强行编造

### 3. 按品类推荐

- 用户指定品类时，从 `categories` 中查找对应商家
- 自动排除 `blacklist` 中的商家
- 如果品类不存在，回复"暂未收录该品类，你可以让我添加"
- 如果品类下所有商家都在黑名单中，告知用户该品类暂无可用商家

### 4. 黑名单管理

用户可以通过自然语言操作黑名单：

- **添加**: "把XX加入黑名单" / "拉黑XX" → 将商家名加入 `blacklist` 数组去重后写回
- **移除**: "把XX从黑名单移除" / "取消拉黑XX" → 从 `blacklist` 数组中删除该商家
- **查看**: "查看黑名单" → 列出现有黑名单

操作完成后必须给出确认回复，并说明已更新配置文件。

### 5. 商家扩充

用户可以通过自然语言添加新商家：

- "添加{品类}新店'{商家名}'" / "新增{商家名}到{品类}" 
- 支持附带推荐理由：如 "添加炒饭新店'老王炒饭'，理由是分量大"
- 如果品类不存在，自动创建该品类
- 操作完成后确认写入

### 6. 默认推荐（用户未指定品类）

当用户只说"今天吃什么"或类似模糊询问时：

1. 先检查当天是否有特殊优惠，有则优先推荐
2. 如果没有优惠，从所有品类中随机挑 2-3 个推荐，避开黑名单
3. 列出可选的品类供用户进一步选择

## 回复风格

- 简洁直接，先给结论再给理由
- 推荐时附带简要推荐理由（来自配置文件）
- 所有交互使用中文
- 示例回复模板：

  有优惠时：
  > 今天是{星期X}，{商家}今天{优惠说明}，强烈推荐！如果想换口味，还可以选{列出2-3个品类}。

  指定品类时：
  > 推荐{商家名}，理由是{推荐理由}。需要我帮你对比其他选择吗？

  品类不存在时：
  > 暂未收录"{品类}"品类。你可以告诉我商家名和推荐理由，我帮你加进去。

## 配置文件操作

所有数据变更通过编辑 `config.json` 实现。修改时：
- 读取当前文件内容
- 按用户意图修改对应字段
- 使用 Edit 工具写入，保持 JSON 格式规范
- 写入后确认变更内容
